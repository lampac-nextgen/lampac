# API Эндпоинты (API)

Lampac NextGen предоставляет HTTP API для взаимодействия с клиентами (в первую очередь с Lampa) и для диагностики состояния сервера.

## 1. Базовые системные эндпоинты (Core)

Эти пути доступны всегда и обрабатываются ядром приложения.

| Метод | Путь | Описание |
| --- | --- | --- |
| `GET` | `/version` | Возвращает текущую версию сервера. |
| `GET` | `/api/myip` | Возвращает публичный IP-адрес клиента, как его видит сервер. |
| `GET` | `/api/headers` | Показывает HTTP-заголовки текущего запроса (полезно для отладки прокси). |
| `GET` | `/api/geo[?ip=]` | Возвращает гео-данные по IP-адресу (определяется через встроенную БД MaxMind GeoLite2). |
| `GET` | `/api/chromium/ping` | Проверка доступности headless браузера Chromium (ответ `pong`). |
| `POST` | `/rch/result?id=` | Эндпоинт Remote Client Hub для записи результата ответа (до 10 МБ). |
| `POST` | `/rch/gzresult?id=`| То же, что и выше, но для Gzip сжатых ответов. |
| `WS`  | `/ws` | Эндпоинт NativeWebSocket для push-уведомлений RCH. |

## 2. Статистика (Diagnostic Stats)
**Внимание:** Для работы этих эндпоинтов в `init.conf` должен быть включен параметр `openstat.enable: true`. Иначе они вернут ошибку 404.

| Метод | Путь | Описание |
| --- | --- | --- |
| `GET` | `/stats/gc` | Статистика сборщика мусора (GC) и использования памяти процессом. *(Доступен всегда)* |
| `GET` | `/stats/request` | Статистика HTTP-запросов: счетчики в минуту/час, количество соединений. |
| `GET` | `/stats/threadpool` | Диагностика пула потоков приложения (ThreadPool). |
| `GET` | `/stats/tempdb` | Использование пулов памяти (RecyclableMemoryStream), состояние кешей. |
| `GET` | `/stats/browser/context` | Данные о процессах Playwright (количество открытых вкладок и контекстов браузера). |

## 3. Ядро VOD (Модуль Online)

| Метод | Путь | Описание |
| --- | --- | --- |
| `GET` | `/online.js` | Отдает сгенерированный JS-плагин VOD для клиента Lampa. |
| `GET` | `/online/js/{token}` | Отдает плагин с привязанным токеном авторизации. |
| `GET` | `/lite/{provider}` | Получение списка контента или стримов конкретного провайдера (например, `/lite/rezka`). |
| `GET` | `/externalids` | Получение маппинга внешних идентификаторов фильмов (TMDB -> Kinopoisk -> IMDB). |

## 4. Ядро 18+ (Модуль SISI)

| Метод | Путь | Описание |
| --- | --- | --- |
| `GET` | `/sisi.js` | JS-плагин взрослого контента. |
| `GET` | `/{provider}` | Получение ленты или видео с платформы 18+ (например, `/phub`). |
| `GET` | `/sisi/bookmark` | Управление закладками SISI. |
| `GET` | `/sisi/history` | Просмотр истории SISI. |
| `GET` | `/nexthub` | Доступ к браузеру сайтов через YAML-конфиги (NextHUB). Требует query-параметр `plugin=...`. |

## 5. Дополнительные модули

| Метод | Путь | Описание |
| --- | --- | --- |
| `GET` | `/ts/...` | Управление и проксирование запросов ко встроенному TorrServer. |
| `GET` | `/tmdb/...` | Проксирование запросов к TMDB с локальным файловым кешированием. |
| `GET` | `/timecode/...` | Сохранение и получение позиций просмотра (TimeCode). |
| `GET` | `/bookmark/...` | Синхронизация общих пользовательских закладок (Sync). |
| `GET` | `/weblog` | Страница отладки. Транслирует логи исходящих запросов и логов браузера через WebSockets. *(Требует пароль root)*. |
