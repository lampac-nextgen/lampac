# Модули и Расширения (Modules)

Lampac NextGen построен по модульной системе. Большая часть функционала (включая парсеры сайтов) вынесена в отдельные модули, которые компилируются на лету или подгружаются в виде библиотек при старте.

## Состояние модулей по умолчанию

Некоторые модули по умолчанию **отключены** (через массив `SkipModules` в `config/base.conf` или через `"enable": false` в их `manifest.json`), чтобы сэкономить ресурсы сервера. Вы можете включить их, удалив из списка `SkipModules` в вашем `init.conf` или изменив манифест.

| Модуль | Включен по умолчанию | Описание |
| --- | :---: | --- |
| **Online** | ✅ Да | Ядро Video-On-Demand (VOD). Обрабатывает эндпоинты `/lite/…` и выдает клиентский скрипт `online.js`. |
| **SISI** | ✅ Да | Ядро контента 18+. Выдает `sisi.js`, управляет историей и закладками. |
| **CubProxy / TmdbProxy** | ✅ Да | Проксирует запросы к TMDB (кеш), ускоряя загрузку постеров и метаданных. |
| **JacRed** | ✅ Да | Агрегатор торрент-трекеров (Rutor, Kinozal, Rutracker и др.). Совместим с API Jackett. |
| **LampaWeb** | ✅ Да | Встроенный веб-хостинг для интерфейса Lampa. Автоматически обновляется с GitHub. |
| **TorrServer** | ✅ Да | Встроенный торрент-клиент. Позволяет смотреть торренты "на лету" без полного скачивания. |
| **TimeCode** | ✅ Да | Сохраняет позицию просмотра видео, позволяя продолжить с того же места на другом устройстве. |
| **NextHUB** | ✅ Да | Каталог 18+ сайтов, работающий на основе YAML описаний. |
| **Catalog** | ⛔ Нет | Каталог сайтов (без экранирования запросов). Только для локальной сети. |
| **DLNA** | ⛔ Нет | DLNA/UPnP медиасервер для просмотра локальных файлов на ТВ. (Требует отключения в `SkipModules`). |
| **Sync / Storage** | ⛔ Нет | Синхронизация закладок и хранилища между устройствами через БД SQLite. |
| **Tracks / Transcoding** | ⛔ Нет | Управление субтитрами и HLS транскодирование через FFmpeg. |
| **AdminPanel** | ⛔ Нет | Встроенная веб-админка. Выключена в `manifest.json`. |

## Как подключить / включить выключенный модуль

Есть два способа контроля загрузки модулей:

1. **`init.conf` (`BaseModule.SkipModules`)**
   Имена модулей, перечисленные в этом массиве, гарантированно игнорируются при загрузке. Чтобы включить `DLNA` или `Sync`, уберите их из этого массива в вашем `init.conf`.

2. **`manifest.json`**
   Каждый модуль в директориях `module/` или `mods/` имеет свой манифест. Если в нем прописано `"enable": false`, модуль не запустится.
   Если вы используете Docker, вы можете создать папку на хосте, положить туда модифицированный `manifest.json` с `"enable": true` и смонтировать её в контейнер (в `/lampac/mods/ИмяМодуля`).

## Создание собственных модулей

Пользователи могут добавлять свой функционал без пересборки всего Lampac. Для этого используется папка `mods/`.

1. Создайте папку в `mods/` (например, `mods/MyCustomModule/`).
2. Внутри создайте файл `manifest.json`:
   ```json
   {
     "name": "MyCustomModule",
     "description": "Мой первый парсер",
     "version": "1.0",
     "enable": true,
     "dynamic": true
   }
   ```
   *Параметр `"dynamic": true` означает, что сервер будет следить за изменениями кода и перекомпилировать его на лету.*
3. Положите рядом C# файлы (`.cs`). Они будут скомпилированы при старте.
4. Вы можете наследоваться от `BaseController` или `BaseOnlineController` для быстрого создания новых источников контента.
