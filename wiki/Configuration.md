# Конфигурация (Configuration)

Настройка сервера осуществляется через файл `init.conf` (или `init.yaml`), который находится в корне рабочей директории сервера (например, `/opt/lampac/init.conf` при нативной установке или смонтирован в `/lampac/init.conf` при использовании Docker).

Сервер поддерживает **горячую перезагрузку**: файл отслеживается каждую секунду, и при его изменении настройки применяются без перезапуска приложения (результат сливается с базовыми настройками `base.conf`).

В этой статье подробно описаны все основные секции конфигурации.

---

## 1. Базовые сетевые настройки (`listen`)

Этот блок отвечает за порт, протокол и общие настройки HTTP-сервера.

```jsonc
{
  "listen": {
    "ip": "0.0.0.0",
    "port": 9118,
    "scheme": "http",
    "https": false,
    "compression": true,
    "ResponseCancelAfter": 15,
    "localhost": "127.0.0.1",
    "version": true
  }
}
```

| Параметр | Описание |
| --- | --- |
| `ip` | IP-адрес для прослушивания соединений (по умолчанию `0.0.0.0` - все интерфейсы). |
| `port` | Порт сервера. |
| `scheme` | Используемая схема: `http` или `https`. |
| `https` | Включить поддержку HTTPS (требует дополнительной настройки сертификатов). |
| `compression` | Включить GZIP/Brotli сжатие для ответов сервера. |
| `ResponseCancelAfter` | Максимальное время ожидания ответа от удаленных серверов (в секундах). |
| `version` | Показывать версию сервера в заголовках или на стартовой странице. |

## 2. Настройки загрузки модулей (`BaseModule`)

Эта секция позволяет управлять тем, какие модули будут загружены при старте.

```jsonc
{
  "BaseModule": {
    "ValidateRequest": true,
    "BlockedBots": true,
    "LoadModules": [".*"],
    "SkipModules": [
      "Catalog",
      "DLNA",
      "JacRed",
      "Sync",
      "TimeCode",
      "TorrServer",
      "Tracks",
      "Transcoding",
      "WebLog"
    ]
  }
}
```

| Параметр | Описание |
| --- | --- |
| `ValidateRequest` | Включает валидацию входящих запросов. |
| `BlockedBots` | Автоматическая блокировка запросов от известных поисковых ботов. |
| `LoadModules` | Массив модулей для загрузки. `[".*"]` означает загрузку всех доступных. Можно указывать маски или названия. |
| `SkipModules` | Список имен модулей, которые **не** будут загружены. Используйте для отключения тяжелых или ненужных вам модулей. |

---

## 3. Настройки ядра Онлайн-кино (`online`)

Настройки для клиентского плагина `online.js` и общего поведения источников.

```jsonc
{
  "online": {
    "name": "Lampac NextGen",
    "spiderName": "Lampac NextGen",
    "version": true,
    "btn_priority_forced": true
  }
}
```

| Параметр | Описание |
| --- | --- |
| `name` | Имя инстанса, которое будет отображаться в интерфейсе Lampa. |
| `spiderName` | Имя "паука" (агрегатора) в UI. |
| `version` | Передавать версию плагина. |
| `btn_priority_forced` | Принудительно сортировать кнопки провайдеров в UI согласно их приоритетам (см. настройку провайдеров ниже). |

## 4. Защита и безопасность (WAF)

Встроенный брандмауэр позволяет ограничивать доступ и предотвращать атаки.

```jsonc
{
  "WAF": {
    "enable": true,
    "countryAllow": ["RU", "UA", "BY", "KZ"],
    "whiteIps": ["192.168.1.0/24"],
    "bruteForceProtection": true,
    "limit_map": {
      "/lite/": 10,
      "/externalids": 10
    }
  }
}
```

| Параметр | Описание |
| --- | --- |
| `enable` | Включение модуля WAF. По умолчанию выключен (`false`). |
| `countryAllow` | Массив кодов стран (ISO 3166-1 alpha-2), которым разрешен доступ. Если массив пуст — разрешены все. |
| `whiteIps` | Белый список IP-адресов или подсетей (CIDR), для которых не применяются ограничения. |
| `bruteForceProtection` | Включение защиты от перебора паролей (для админки и авторизации). |
| `limit_map` | Ограничение количества запросов в секунду (Rate Limit) для конкретных путей. |

## 5. Аутентификация (accsdb)

Модуль `accsdb` позволяет ограничить доступ к серверу только для авторизованных пользователей с уникальными идентификаторами (UID).

```jsonc
{
  "accsdb": {
    "enable": true,
    "accounts": "user1:2026-12-31,user2:2027-06-01",
    // Альтернативный, более удобный формат:
    "users": [
      { "login": "user1", "expired": "2026-12-31" },
      { "login": "user2", "expired": "2027-06-01" }
    ]
  }
}
```

| Параметр | Описание |
| --- | --- |
| `enable` | Включает систему аккаунтов. |
| `accounts` | Простой строковый формат `логин:дата_истечения`. |
| `users` | Массив объектов с пользователями и датой окончания подписки. |

## 6. Playwright (Обход JS-защит, Cloudflare)

Для работы с источниками, которые защищены сложными скриптами (например, Cloudflare), сервер использует реальный браузер (Chromium/Firefox) в headless режиме через библиотеку Playwright.

```jsonc
{
  "chromium": {
    "enable": true,
    "executablePath": "/usr/bin/chromium",
    "count": 1,
    "restart": 3600
  },
  "firefox": {
    "enable": false,
    "count": 1
  }
}
```

| Параметр | Описание |
| --- | --- |
| `enable` | Включить использование браузера. |
| `executablePath` | Путь к бинарному файлу браузера (уже установлен в Docker-образе). |
| `count` | Количество одновременных процессов браузера (контекстов). |
| `restart` | Время в секундах, через которое браузер будет автоматически перезапущен для очистки памяти. |

## 7. Контент 18+ (SISI)

Настройки для модуля взрослого контента.

```jsonc
{
  "sisi": {
    "lgbt": false,
    "spider": false,
    "NextHUB": {
      "sites_enabled": "pornhub,beeg"
    },
    "history": {
      "enable": false
    }
  }
}
```

| Параметр | Описание |
| --- | --- |
| `lgbt` | Включать ли контент соответствующей тематики в результатах. |
| `spider` | Использовать ли агрегацию нескольких источников. |
| `NextHUB.sites_enabled` | Разрешает загружать из папки `sites/` только те YAML-конфиги, в именах которых содержатся указанные слова (через запятую). Если удалить строку, будут загружены все 18+ провайдеры. |
| `history.enable` | Включить ведение локальной истории просмотров в SQLite. |

## 8. Модуль LampaWeb (Встроенный интерфейс)

Позволяет тонко настраивать плагины, которые будут загружены в интерфейсе Lampa "из коробки", когда пользователь обращается к `index.html` вашего хоста.

```jsonc
{
  "LampaWeb": {
    "initPlugins": {
      "pirate_store": true,
      "jacred": true,
      "tmdbProxy": true,
      "cubProxy": true,
      "online": true,
      "sisi": true,
      "torrserver": true,
      "timecode": true
    }
  }
}
```

| Параметр | Описание |
| --- | --- |
| `pirate_store` | Загружать базовый магазин Lampa. |
| `tmdbProxy` | Использовать ваш встроенный кеширующий сервер для получения постеров. |
| `online` | Включает в клиенте VOD провайдеров. |
| `sisi` | Включает 18+ контент. |

## 9. Авторизация через Telegram (Community Modules)

Альтернатива встроенному `accsdb` с паролями. Позволяет пользователям входить на сервер через Telegram-бота.
*(Требуется удалить `TelegramAuth` и `TelegramAuthBot` из `BaseModule.SkipModules`)*

```jsonc
{
  "TelegramAuth": {
    "enable": true,
    "data_dir": "database/tgauth", // Папка БД
    "mutations_api_secret": "ВАШ_ДЛИННЫЙ_УНИКАЛЬНЫЙ_СЕКРЕТ_123",
    "owner_telegram_ids": [123456789], // Ваш ID в Telegram
    "max_active_devices_per_user": 5 // Лимит устройств
  },
  "TelegramAuthBot": {
    "enable": true,
    "bot_token": "ТОКЕН_БОТА_ОТ_BOTFATHER",
    "lampac_base_url": "http://127.0.0.1:9118", // Внутренний или внешний адрес сервера
    "service_display_name": "Мой Lampac Bot",
    "mutations_api_secret": "ВАШ_ДЛИННЫЙ_УНИКАЛЬНЫЙ_СЕКРЕТ_123", // Должен совпадать с TelegramAuth!
    "notify_admins_on_pending_provision": true
  }
}
```

## 10. Настройка провайдеров контента (VOD)

Каждый провайдер настраивается в отдельном блоке, название которого совпадает с именем провайдера (с учетом регистра).

```jsonc
{
  "Rezka": {
    "enable": true,
    "host": "https://rezka.ag",
    "priority": 1
  },
  "Filmix": {
    "enable": true,
    "host": "https://filmix.biz",
    "token": "ВАШ_ТОКЕН",
    "priority": 2
  },
  "KinoPub": {
    "enable": true,
    "token": "ВАШ_ТОКЕН"
  },
  "Kodik": {
    "enable": true,
    "token": "ВАШ_ТОКЕН",
    "priority": 3
  }
}
```

| Параметр | Описание |
| --- | --- |
| `enable` | Включение/отключение парсинга данного провайдера. |
| `host` | Зеркало или актуальный адрес сайта провайдера (важно для обхода блокировок). |
| `token` | Ваш личный API-токен для доступа к провайдеру (если требуется). |
| `priority` | Приоритет отображения кнопки в клиенте Lampa (чем меньше цифра, тем выше кнопка). Работает совместно с `online.btn_priority_forced`. |

## 11. Дополнительные технические настройки

- **Remote Client Hub (RCH)**
  ```jsonc
  {
    "rch": {
      "enable": false,
      "requiredConnected": 1
    }
  }
  ```
  *WebSocket-реле для клиентов за NAT. Позволяет устройствам связываться друг с другом через ваш сервер.*

- **Кеш (`cache`)**
  ```jsonc
  {
    "cache": {
      "extend": 180 // Время продления жизни кеша в минутах
    }
  }
  ```

- **Статистика (`openstat`)**
  ```jsonc
  {
    "openstat": {
      "enable": true // Включает эндпоинты статистики (например, /stats/request)
    }
  }
  ```

- **Сборщик мусора и память (`GC`)**
  ```jsonc
  {
    "GC": {
      "Concurrent": true,
      "ConserveMemory": 0,
      "HighMemoryPercent": 90, // Порог использования памяти до агрессивной сборки
      "RetainVM": false
    }
  }
  ```

- **Kit (Шифрование)**
  ```jsonc
  {
    "kit": {
      "aesgcmkeyName": "" // Имя ключа для шифрования потоков
    }
  }
  ```

- **TMDB Прокси (`tmdb`)**
  ```jsonc
  {
    "tmdb": {
      "cache_img": 60 // Время кеширования изображений с TMDB в минутах
    }
  }
  ```

- **Cub Прокси (`cub`)**
  ```jsonc
  {
    "cub": {
      "scheme": "https",
      "domain": "cub.red",
      "api_key": "ВАШ_CUB_API_КЛЮЧ"
    }
  }
  ```

- **Прокси сервера (`serverproxy`)**
  ```jsonc
  {
    "serverproxy": {
      "verifyip": false,
      "image": {
        "cache_time": 60 // Время хранения кешированных изображений
      }
    }
  }
  ```

- **Статические файлы (`Staticache`)**
  ```jsonc
  {
    "Staticache": {
      "enable": true, // Включить кеширование статики
      "max_mb": 100 // Максимальный размер папки статики в MB
    }
  }
  ```
